# Настройка поля play_icon в Supabase

## Проблема

При сохранении трека возникает ошибка 400, потому что поле `play_icon` не существует в таблице `tracks` в Supabase.

## Решение

Нужно добавить поле `play_icon` в таблицу `tracks` в Supabase.

### Шаги:

1. Откройте Supabase Dashboard → Table Editor → `tracks`

2. Нажмите "Add Column" (или "Добавить колонку")

3. Настройте новое поле:
   - **Name (Имя)**: `play_icon`
   - **Type (Тип)**: `text` или `varchar`
   - **Default value (Значение по умолчанию)**: `NULL` (или оставьте пустым)
   - **Is Nullable (Может быть NULL)**: ✅ Да (включено)
   - **Is Unique (Уникальное)**: ❌ Нет (выключено)

4. Нажмите "Save" (Сохранить)

### SQL (альтернативный способ):

Если предпочитаете SQL, выполните в SQL Editor:

```sql
ALTER TABLE tracks 
ADD COLUMN play_icon TEXT DEFAULT NULL;
```

### После добавления поля:

1. Откройте файл `src/features/artist/ArtistTracks.jsx`
2. Найдите строки с комментарием `// ВРЕМЕННО: не отправляем play_icon`
3. Раскомментируйте код для отправки `play_icon`:

```javascript
// Раскомментируйте эти строки:
if (data.play_icon !== undefined && data.play_icon !== null) {
  updateData.play_icon = data.play_icon;
}
```

4. Удалите или закомментируйте строку `// ВРЕМЕННО: не отправляем play_icon`

### Проверка:

После добавления поля и обновления кода:
- Сохранение трека должно работать без ошибок
- Выбранная иконка плеера должна сохраняться в БД
- На странице трека должна отображаться выбранная иконка
